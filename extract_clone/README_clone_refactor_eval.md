

# Clone Refactorability Evaluation Suite

## The Refactorability Metric

For a clone pair to be considered strictly refactorable (`Refactorable = 1`), it must satisfy both structural and type-level equivalence. The script enforces the following logic:

$$\text{Refactorable} = (\text{InCount}_L == \text{InCount}_R) \land (\text{OutCount}_L == \text{OutCount}_R) \land (\text{InType}_L == \text{InType}_R) \land (\text{OutType}_L == \text{OutType}_R)$$

Empty or missing inputs/outputs are standardized to `N/A` to ensure schema consistency for downstream Pandas/CSV analysis.

## Requirements

* **Python 3.7+**
* `pandas` and `scikit-learn` (Required only for the downstream analysis script)

## Step 1: Generating the Evaluation Dataset (`clone_refactorability_evaluation.py`)

This script processes the raw JSONL and prediction files to create the unified CSV dataset.

### Command-Line Arguments

| Argument | Type | Description |
| --- | --- | --- |
| `--jsonl` | Required | Path to the metadata JSONL file containing AST-extracted function signatures. |
| `--gt` | Required | Path to the ground truth text file (e.g., BigCloneBench `test.txt`). |
| `--pred` | Required | Path to the model predictions **BEFORE** adaptation. |
| `--pred_adapt` | Required | Path to the model predictions **AFTER** adaptation. |
| `--out` | Required | Destination path for the generated CSV analysis report. |

### Usage Example

Assuming you are running this from your `~/research_dream/llm-clone/extract_clone` directory, here are the exact absolute paths you should use:

```bash
python clone_refactorability_evaluation.py \
  --jsonl "/home/user1-system11/research_dream/llm-clone/extract_clone/output/extractable_nicad_block_clones/camel_clone_analysis.jsonl" \
  --gt "/home/user1-system11/research_dream/llm-clone/Task/Clone-Detection-BigCloneBench/dataset/nicad_block_java/camel/test.txt" \
  --pred "/home/user1-system11/research_dream/llm-clone/Task/Clone-Detection-BigCloneBench/codebert/saved_models_bcb/predictions_bcb_java_block_function/predictions_camel_test.txt" \
  --pred_adapt "/home/user1-system11/research_dream/llm-clone/Task/Clone-Detection-BigCloneBench/codebert/saved_models_combined/predictions_bcb_java_block_function/predictions_camel_test.txt" \
  --out "/home/user1-system11/research_dream/llm-clone/extract_clone/output/clone_refactorability_evaluation/camel_refactor_clone_eval.csv"

```

## Step 2: Downstream Impact Analysis (`analyze_refactor_impact.py`)

Once the evaluation dataset is generated, use the `analyze_refactor_impact.py` script to calculate the concrete performance gains achieved by model adaptation.

This script computes:

* **Overall Precision, Recall, and F1 Scores** (Before vs. After adaptation).
* **Refactorability-Aware Metrics**: F1 scores specifically for the subset where `Refactorable == 1`.
* **Adaptation Recovery Rates**: Quantifies how many missed refactoring opportunities (False Negatives) were successfully "fixed" by the adaptation process, as well as any regressions.

### Command-Line Arguments

| Argument | Type | Description |
| --- | --- | --- |
| `--csv` | Required | Path to the CSV file generated by `clone_refactorability_evaluation.py`. |
| `--log` | Optional | Path to save the terminal output as a plain text log file for research records. |

### Usage Example

To run the analysis and print the results to the terminal:

```bash
python3 analyze_refactor_impact.py \
  --csv "/home/user1-system11/research_dream/llm-clone/extract_clone/output/clone_refactorability_evaluation/camel_refactor_clone_eval.csv"

```

To run the analysis and simultaneously save a permanent log file:

```bash
python3 analyze_refactor_impact.py \
  --csv "/home/user1-system11/research_dream/llm-clone/extract_clone/output/clone_refactorability_evaluation/camel_refactor_clone_eval.csv" \
```
